# Lens

**全能型 Emby 媒体库深度管理与自动化维护工具箱**

Lens 是一款专为私有云媒体服务器设计的综合管理平台。它集成了元数据净化、演员维护、智能去重、深度搜索及 Webhook 联动等多项原子化工具，旨在为 Emby 用户提供一个高效、透明且自动化的运维环境。

---

## 🛠️ 核心功能矩阵 (基于原子化导航)

Lens 的每一项功能都以独立工具的形式存在，你可以通过侧边栏快速切换：

### 📊 管理仪表盘 (Dashboard)
*   **概览统计**：直观展示媒体库的资源总量、库数量、系统运行状态。
*   **审计流水**：记录最近的维护操作，确保系统变动可追溯。

### 📈 播放统计报表 (Playback Reporting)
*   **全景仪表盘设计**：前端视觉，采用沉浸式毛玻璃质感与全景数据视角。
*   **Netflix 风格海报墙**：热门电影与剧集排行采用带有 **3D 悬停特效、金色/银色排名角标** 及详情浮层的海报墙展示。
*   **领奖台模式排行**：前三名拥有专属领奖台位，配套皇冠 👑 与动态成就勋章（肝帝、修仙党等）。
*   **深度数据对标**：基于 Emby `Playback Reporting` 插件，全接口对标桌面端调试工具，提供精准的播放数据透视。
*   **实时活动流水**：动态展示最近的播放活动，支持 **季集详情 (SxxExx)** 识别与系列海报自动关联。
*   **核心技术保障**：
    -   **后端图片代理**：通过 Lens 后端中转图片请求，彻底解决浏览器直连 Emby 导致的跨域、权限及 404 问题。
    -   **智能 ID 寻址**：自动将插件的内部数字 ID 转换为 Emby Guid，确保海报 100% 正确显示。
*   **智能自动刷新**：支持自定义刷新频率（1s/5s/10s 等），0 为禁用，适合作为大屏监控看板。

### 📅 Emby 任务计划 (Scheduled Tasks)
*   **原生风格增强**：深度还原 Emby 官方任务列表布局，兼具清晰度与操作便捷性。
*   **自动分类隔离**：根据任务属性（Library, System, Danmu 等）自动分组，并配备彩色装饰线与专属图标。
*   **实时进度监控**：任务执行时自动开启 **呼吸灯动效与实时进度条**，支持 5 秒级状态自动同步。
*   **全面控制能力**：支持在 Lens 界面内一键远程启动挂起任务或强制停止正在运行的异常进程。

### 🌐 多 Emby 服务端管理 (Multi-Server Support)
*   **多实例纳管**：支持同时配置并管理多个 Emby 服务器，每个服务器拥有独立的连接凭据与会话令牌。
*   **一键快速切换**：在侧边栏集成全局服务器切换器，无需刷新页面即可在不同 Emby 实例间秒级切换，所有工具（查重、打标、净化等）将自动作用于当前激活的服务器。
*   **解耦式配置弹窗**：提供独立的服务器管理界面，支持连通性实时测试、登录鉴权及 Emby 真实 ServerId 的自动识别。
*   **Webhook 智能路由**：自动标签 Webhook 能够基于 Payload 中的 `ServerId` 自动匹配本地配置，支持多个 Emby 实例同时向 Lens 发送自动化指令而互不干扰。

### 🏠 站点导航页 (Site Navigation)
*   **沉浸式管理**：支持在导航页直接进行管理。**鼠标右键**点击卡片即可编辑或删除站点，**悬停**在分类标题上即可快速添加站点。
*   **高级视觉定制 (Style & Aesthetic)**：
    *   **SunPanel 灵感设计**：内置类似 SunPanel 的精致透明毛玻璃质感，支持卡片背景、边框、文字颜色及高斯模糊度的像素级微调。
    *   **动态背景系统**：
        - **必应每日壁纸 (Bing Wallpaper)**：支持接入必应官方每日壁纸。提供全球 5 个地区（中、美、日、德、英）切换，支持查看过去 7 天的历史壁纸，并支持 **4K UHD** 超高清画质。
        - **壁纸故事**：支持在页面角落展示当前壁纸的标题、摄影地及版权故事。
        - **自定义上传**：支持本地上传背景图，并可调整透明度与模糊度。
    *   **每日一言 (Hitokoto)**：深度集成 [Hitokoto](https://hitokoto.cn/)。支持在页头展示精美的文字句子，点击即可实时刷新，并可自定义显示位置。
*   **高度自由布局 (Layout)**：
    *   **对齐与边距**：支持主标题、副标题及一言的水平对齐（左、中、右）切换。支持调整内容最大宽度、顶部边距（MT）及项目间距。
    *   **文本自定义**：支持自定义页面主标题与操作提示文字，支持留空以实现极简视觉。
*   **沉浸模式切换**：右上角集成导航栏切换按钮，支持一键隐藏/显示侧边导航。
*   **HD-Icons 图标库集成**：深度集成 [xushier/HD-Icons](https://github.com/xushier/HD-Icons) 1700+ 精美图标。支持在线挑选，涵盖多种风格。
*   **智能拖拽排序**：支持对站点卡片和分类区块进行物理拖拽排序。
*   **本地图标缓存**：自动下载并缓存站点图标至本地，解决防盗链与离线访问问题。
*   **全量备份与恢复**：支持导出包含配置与物理图标文件的 .zip 压缩包。
*   **快捷访问**：支持通过 `/home` 路径（如 `http://ip:6565/home`）无视页面记忆直接进入导航页，适合作为浏览器首页。

### 🐳 Docker 深度运维管理 (Docker Manager)
*   **多机纳管**：支持 **SSH** 管理多台远程服务器，支持主机配置的持久化记忆与编辑。
*   **容器生命周期**：支持容器启动、停止、重启、删除及实时日志查看；状态显示已全面中文本地化。
*   **交互式终端 (Terminal)**：集成 **xterm.js**，支持一键进入容器内部终端，预设 `bash`、`sh`、`ash` 多种 Shell 方案，支持全双工实时交互。
*   **镜像更新检测 (Image Update Detection)**：
    *   **指纹级比对**：基于 OCI 标准的 `Digest` 指纹比对技术（非简单版本号对比），即使是 `latest` 标签也能精准发现更新。
    *   **广泛兼容**：不仅支持 Docker Hub，还通过 OCI 挑战认证机制完美兼容 `lscr.io` (LinuxServer)、`ghcr.io` (GitHub)、`quay.io` 等第三方镜像仓库。
    *   **按需探测**：支持在容器列表对单个镜像进行精准实时探测，规避因批量请求导致的仓库 IP 封锁风险。
*   **自动化运维 (Lens Watchtower)**：
    *   **全自动更新任务**：支持为特定容器开启“自动更新”托管。系统内置调度器，每日凌晨 **03:00** 自动执行检测、拉取及容器重构。
    *   **即时进度通知**：自动更新完成后，通过 Telegram Bot 推送包含主机名、容器名及镜像信息的详细升级报告。
    *   **无感知架构**：基于纯 SSH 穿透实现，无需在宿主机挂载 `docker.sock` 或安装任何 Agent，保持系统极致安全。
*   **环境检测与修复**：
    *   **实时检测**：自动探测远程主机的 Docker 版本、Compose 版本、OS 内核及服务运行状态。
    *   **一键修复/安装**：支持在远程主机上一键自动化安装 Docker 及 Compose 环境。
    *   **灵活代理**：安装过程支持自定义 **HTTP/HTTPS 代理**，并内置阿里云镜像加速，确保极速部署。
    *   **核心服务控制**：支持在网页端直接对远程主机的 Docker 守护进程执行启动、停止、重启操作。
*   **智能更新 (Update)**：一键拉取最新镜像并自动重构容器。
    *   **配置保全**：重构时自动识别并保留高级挂载传播属性（如 `rslave`, `rshared`），确保 CloudDrive2/Rclone 等挂载环境在更新后依然稳定。
*   **Compose 管理**：内置管理器，支持 YAML 在线编辑、目录扫描、项目记忆及按名称固定排序。
    *   **一键远程备份 (Relay Backup)**：支持将远程 Compose 项目文件夹整体打包并拉取回 Lens 本地进行中转存档。
    *   **任意文件夹 SSH 备份**：支持在文件浏览器中将任意远程文件夹（配置/数据目录）一键设为 SSH 自动备份任务，不再局限于 Compose 目录。
    *   **实时语法校验**：集成 `js-yaml`，编辑时自动进行语法检测并提供**中文错误提示**（含行号、列号）。
*   **批量操作**：支持一键对主机下所有项目执行“批量启动/更新”或“批量停止”操作。
*   **配置中心 (Configuration)**：
    *   **Daemon 维护**：可视化配置 `/etc/docker/daemon.json`，包括镜像加速器 (`registry-mirrors`)、HTTP/HTTPS 代理 (`proxies`)、日志切割规则及 `live-restore`。
    *   **双重备份**：修改配置前自动生成远程 `.bak` 文件并同步备份至本地 `data` 目录。
    *   **一键清理**：支持清理未标签镜像 (Dangling)、未使用镜像 (Unused) 及构建缓存。
*   **便捷跳转**：自动解析端口映射，针对 Host 模式自定义端口实现一键访问。

### 🚀 Docker 镜像构建与推送 (Image Builder)
*   **乐高模块化管理**：支持创建多个构建项目，为每个项目配置独立的构建上下文、Dockerfile 路径及目标平台。
*   **跨平台多架构构建 (Multi-arch)**：
    *   **Buildx 深度集成**：支持同时构建并推送 `amd64`、`arm64`、`arm/v7` 等多种架构的镜像。
    *   **一键环境修复**：内置 QEMU 仿真自动安装与专用构建器（`docker-container` 驱动）初始化逻辑，即使是纯净系统也能秒变多架构构建工厂。
*   **多机指挥中心**：支持通过 **SSH** 指定任意远程主机执行构建任务，实现资源调度自动化。
*   **私有仓库全兼容**：
    *   **动态信任机制**：自动识别 HTTP 仓库地址，实时生成并挂载 BuildKit 信任配置，彻底解决内网 IP 仓库的 SSL 协议握手失败问题。
    *   **凭据治理**：支持维护多个镜像仓库的登录凭据，支持连通性实时测试与协议预检。
*   **极致构建加速**：
    *   **远程缓存优化**：自动启用 `--cache-from=type=registry`，支持从远程仓库拉取 Inline 缓存层，极大提升二次构建速度。
    *   **代理加速通道**：支持配置独立的构建代理，并支持将代理配置永久注入构建引擎，解决拉取基础镜像（FROM）时的网络瓶颈。
*   **工业级日志与监控**：
    *   **后台静默执行**：点击构建后任务自动进入后台，不占用前端资源。
    *   **结构化历史记录**：保留详尽的构建历史，支持按版本（Tag）回溯查看每一次任务的完整输出日志。
    *   **多维度即时报告**：构建完成后自动向 Lens 控制台、系统桌面及 Telegram Bot 推送详尽的构建报告（含镜像名、Tag、耗时及状态）。

### 🖥️ 独立系统终端管理 (Terminal Manager)
*   **多会话持久化 (Multi-Session)**：支持同时开启多个 SSH 会话，采用 `v-show` 机制确保切换主机时终端不断开、进程不中断。
*   **安全认证体系**：支持标准的密码认证及 **RSA/ED25519 密钥认证**，支持加密私钥的 Passphrase 自动解密。
*   **快速指令集 (Quick Commands)**：
    *   **可编辑预设**：支持存储常用的运维指令，支持分类编辑与管理。
    *   **灵活执行模式**：支持“点击即执行”或“仅输入不回车”双模式，提升生产环境安全性。
*   **工业级 UI 交互**：
    *   **自适应布局**：采用 Flex 弹性布局及 xterm-addon-fit，完美填满屏幕，自动同步窗口 Resize。
    *   **主题化渲染**：终端配色实时跟随系统主题（暗夜紫、深海蓝等），告别生硬的黑盒体验。
    *   **操作反馈**：内置一键清屏、重连及会话状态实时监测。

### 🐘 PostgreSQL 运维管理 (PostgreSQL Manager)
*   **多实例纳管**：支持同时维护多个 PostgreSQL 服务器配置，支持持久化记忆、**在线编辑**与一键切换。
*   **数据库全生命周期**：
    -   **概览展示**：直观列出所有非模板数据库及其所有者 (Owner) 与备注说明。
    -   **编辑维护**：支持在线修改数据库所有者及添加/更新数据库注释 (Comment)。
    -   **安全管理**：创建数据库时支持指定所有者；删除库时自动执行会话强踢逻辑，确保运维无阻碍。
*   **备份与还原 (Backup & Restore)**：
    -   **逻辑备份**：集成 `pg_dump`，支持一键生成 `.bak` 格式（自定义压缩格式）的物理隔离备份文件。
    -   **自动化联动**：深度集成至“数据备份管理”系统，支持创建定时备份任务，实现无人值守的数据库安全维护。
    -   **极速还原**：集成 `pg_restore`，支持从历史备份列表选择文件进行还原。还原过程自动化处理：强制断开连接 -> 重建数据库 -> 恢复数据。
    -   **文件管理**：支持对备份文件进行管理与删除，所有文件持久化存储于 `data/backups/pg`。
*   **专业级角色管理**：
    -   **精简视图**：自动过滤系统级角色 (pg_*)，仅展示业务相关角色。
    -   **全量权限控制**：支持对角色进行 `LOGIN`、`SUPERUSER`、`CREATEDB`、`CREATEROLE`、`INHERIT`、`REPLICATION` 及 `BYPASSRLS` 的精细化勾选。
    -   **连接与安全**：支持设置 `CONNECTION LIMIT` 及 `VALID UNTIL`（账户有效期），并提供便捷的密码重置功能。
*   **数据浏览器 (Data Browser)**：
    -   **多库穿透**：支持在同一主机下自由切换数据库以浏览不同库的表结构。
    -   **动态网格**：自动解析表结构，根据 `columns` 定义动态生成数据网格，支持查看普通表、视图及外表。
        - **工业级分页**：支持大规模数据分页加载，确保极速响应。
    
### 📦 数据备份管理 (Backup Manager)
*   **跨主机备份 (Remote SSH Backup)**：基于 SSH + SFTP 逻辑，实现“远程打包 -> 异步拉取 -> 本地存档”的完整中转流程，支持将远程服务器数据备份至 Lens 本地存储（或挂载的 NAS/网盘）。
*   **多模式备份支持**：
    *   **7z 压缩**：支持 1-9 级压缩强度调节，支持 AES-256 文件名及内容加密。
    *   **Tar 打包**：标准的 Linux 打包模式，具备极高的兼容性。
    *   **物理同步 (Sync)**：提供**镜像 (Mirror)** 与 **增量 (Incremental)** 两种策略。
    *   **PostgreSQL 自动备份**：联动“PostgreSQL 运维管理”，支持对多个数据库实例进行定时逻辑导出与归档。
*   **硬件感知优化**：
    *   **SSD 模式**：开启全火力多线程并行处理，压榨极致带宽。
    *   **HDD 保护**：自动限制并发线程（max=2），减少磁头频繁寻址，保护机械硬盘寿命。
    *   **云盘优化 (Cloud)**：针对 Rclone/CloudDrive2 挂载点，采用“**本地中转 -> 顺序上传**”逻辑，自动启用 `--size-only` 策略，规避 API 频率限制并防止 FUSE 崩溃。
*   **智能过滤引擎 (Smart Filtering)**：
    *   **脚本级逻辑同步**：备份深度集成 `BackupFilter` 正则过滤引擎。
    *   **清单打包技术**：支持先扫描再过滤，生成精准白名单后再执行打包，确保远程备份忽略规则（如 `.git`, `node_modules`）与本地完全一致。
*   **自动化运行计划**：支持“每天固定时间”、“固定间隔频率”及自定义 Cron 表达式。
*   **安全还原体系**：支持“覆盖还原”与“清空还原”双模式，提供备份历史记录溯源。
    
### ♻️ 重复项清理 (Dedupe Ultimate)
*   **多维度查重**：精准识别重复的电影、剧集。
*   **智能优选**：基于分辨率、编码、动态范围自动评估版本优劣。
*   **安全删除**：支持一键清理冗余文件，释放存储空间，并自动更新 Emby 索引。

### 🏷️ 自动标签助手 (Auto Tags)

*   **实时 Webhook**：支持联动 Emby Webhook，新媒体入库时秒级完成自动化打标。
*   **透明工作流**：详尽展示 TMDB 数据抓取与规则匹配过程，拒绝黑箱操作。

### 🔀 类型映射管理 (Type Manager)
*   **标签净化**：将 Emby 原始杂乱的流派（Genre）映射为统一、干净的展示标签。

### 🧹 媒体净化清理 (Cleanup)
*   **批量剔除**：一键移除特定的标签、流派或无效的演员信息，保持媒体库整洁。

### 🔒 元数据锁定器 (Lock Manager)
*   **状态控制**：批量锁定或解锁媒体字段（如标签、封面），防止 Emby 扫描时覆盖手动修改的内容。

### 🔍 项目元数据查询 (Item Query)
*   **深度透视**：直接调取 Emby 内部原始 JSON 响应，方便开发者调试与检查底层元数据。

### 🎯 剧集 TMDB 反查 (Reverse Lookup)
*   **溯源追踪**：根据单集文件 ID，自动定位其所属剧集的 TMDB 绑定状态。

### 🔎 TMDB ID 深度搜索 (Deep Search)
*   **跨类型索引**：全库扫描匹配指定 TMDB ID，支持快速定位特定资源的存储位置。

### 🧪 TMDB 实验中心 (TMDB Lab)
*   **全语言深度抓取**：直接对接 TMDB 官方 API，支持指定语言抓取或全语言 (All Translations) 模式抓取。

*   **全量标题与别名池**：自动整合主标题、中文标题，并从 Infobox 复杂的嵌套结构中实时提取所有“别名/又名”，为自动化分拣提供精准依据。
*   **角色与章节透视**：提供横向滚动的角色列表（含头像与主角/配角身份）及详尽的章节详情预览，支持原始 JSON 全量数据导出。

*   **全方位身份探测**：集成生日、逝世日期、出生地及社交 ID (IMDB/TMDB) 的深度审计，并支持原始 JSON 全量数据导出。

### 🧪 AI 实验室 (AI Lab)
*   **通用 LLM 接入**：支持主流大语言模型接入，提供灵活的“乐高式”模块化架构，作为全系统 AI 能力的孵化器。

### 🔖 书签同步管理 (Bookmark Management)
*   **跨平台整合**：支持从浏览器导入 HTML 书签文件，实现多设备书签的统一存储与管理。
*   **AI 智能整理 (AI Organize)**：
    *   **全自动规范化**：AI 自动识别并重命名书签标题，剔除冗余后缀（如“- 百度搜索”），让书签列表前所未有的整洁。
    *   **智能分类搬家**：深度分析书签内容与 URL 语义，自动创建合理的文件夹结构并将书签精准归类。
    *   **递归深度清理**：任务完成后自动开启“剥洋葱”式扫描，彻底清理所有变空的旧文件夹残骸。
    *   **流式日志监控**：采用非阻塞后台处理模式，支持在浏览器控制台实时查看 AI 的“搬家直播”记录。
*   **智能体检中心**：支持一键对全库书签进行死链检测、重复项扫描，并提供批量清理与合并功能。

### 👤 账号安全管理 (Account Manager)
*   **全局认证开关**：支持一键开启或关闭前端登录验证。关闭后系统进入“免密模式”，适合可信内网环境；开启后提供工业级的登录保护。
*   **多因素认证 (2FA)**：内置双重验证支持，可绑定 Google Authenticator 或 Microsoft Authenticator，为管理员账户提供银行级的安全防护。
*   **凭据维护**：支持在线修改管理员密码，并实时同步认证状态。

### 👥 Emby 用户管理 (Emby User Management)

*   **精细化策略控制**：全量对标 Emby 底层 JSON 配置，支持图形化控制“转码权限”、“下载限制”、“隐藏用户”、“同时播放数”及“比特率限制”等 30+ 项策略。
*   **配置备份与一键克隆**：支持将单个或全部用户的权限策略备份为本地 JSON 文件。通过还原功能，可实现将一个服务器的用户配置秒级“克隆”到新服务器，还原过程支持自动创建同名用户并注入所有设置。

### 📁 Emby 媒体库管理 (Emby Library Management)
*   **极速建库方案**：主要用于 **新服务器的媒体库初始化**。支持快速新增库、配置类型及文件夹路径。

*   **全量快照恢复**：支持对媒体库配置进行本地 JSON 存档。在新服务器上点击“还原”，即可自动重建媒体库结构并应用所有高级刮削参数。

为了提升“站点导航”的使用体验，Lens 提供了一个专门的浏览器扩展程序（位于 `lens-new-tab-extension` 目录）：

*   **一键收藏站点**：点击浏览器右上角插件图标，自动抓取当前页面的标题、URL 和图标。支持预览图标、修改标题及选择 Lens 导航分类。
*   **智能图标持久化**：插件抓取时会自动联动 Lens 后端，将站点图标下载并缓存至您的服务器，解决图标防盗链问题。
*   **新标签页接管 (New Tab Override)**：支持将浏览器的“新标签页”替换为您的 Lens 导航页。
    -   **极速跳转**：采用 Chrome 原生 API 实现毫秒级重定向，地址栏将直接显示您的 Lens IP 路径。
    -   **灵活配置**：内置开关，支持一键设置新标签页是进入“纯 IP 根目录”还是直接进入“/home 导航模式”。

#### 📦 如何安装插件：
1.  打开 Chrome 浏览器，进入 `chrome://extensions/`。
2.  开启右上角的“**开发者模式**”。
3.  点击“**加载已解压的扩展程序**”，选择本项目中的 `lens-new-tab-extension` 文件夹。
4.  点击插件图标 -> 齿轮图标，配置您的 Lens 服务器地址（如 `http://192.168.50.202:6565`）。

Lens 作为一个深度集成媒体服务器权限的管理工具，虽然内置了完善的账号密码体系及 2FA 多因素认证，但我们仍强烈建议遵循以下安全准则：

1.  **内网环境运行**：建议将 Lens 运行在受信任的家庭/企业局域网内。
2.  **避免公网暴露**：**极其不建议**通过路由器端口转发将 Lens 直接暴露在公网环境下。

## 🚀 快速部署

### 方式一：镜像直接部署 (推荐)
如果你不需要修改代码，直接使用 Docker Hub 上的预构建镜像：

1.  **下载配置**：保存以下内容为 `docker-compose.yml`：
    ```yaml
    version: '3.8'
    services:
      lens:
        image: pipi20xx/lens:latest
        container_name: lens
        network_mode: bridge
        ports:
          - "6565:6565"
        volumes:
          - ./data:/app/data
        environment:
          - DATABASE_URL=sqlite+aiosqlite:////app/data/lens.db
          - TZ=Asia/Shanghai
        restart: always
    ```
2.  **启动服务**：`docker-compose up -d`
3.  **访问系统**：浏览器打开 `http://localhost:6565`

### 方式二：源码编译部署
如果你需要进行二次开发或自定义：

1.  **克隆代码**：`git clone ...`
2.  **启动服务**：`docker-compose up -d --build`

### 🛠️ 初始配置步骤
1.  **安全设置**：首次部署建议前往“账号安全管理”修改默认密码 (`admin123`) 并建议开启 2FA。
2.  **环境集成**：前往“系统设置”，通过“添加新服务器”配置一个或多个 Emby 实例。
3.  **刷新索引**：进入对应工具（如重复项清理），点击“从 Emby 同步”。
4.  **服务器切换**：通过侧边栏左上角的下拉菜单，在多个 Emby 实例间自由切换。
5.  **外部对接**：如需外部脚本调用，在“外部控制体系”生成 API Token。
